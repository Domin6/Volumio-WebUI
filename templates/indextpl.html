            You can perform several kind of searches by typing text. All methods are case-insensitive.

            • Full-text search for any term, in genres, artists, albums or songs. E.g: Highway hell will return both items that contain "highway" or "hell". For instance, you'd get ACDC's "Highway to Hell", but also Deep Purple's "Highway Star" and No Doubt's "Hella Good".

            • Search for exact terms in genres, artists, albums or songs by using double-quotes. E.g: "Highway to hell" (notice the double-quotes) will return only songs with this exact name.

            • You can restrict filters with advanced patterns, by typing {<word><op><value>} where <word> is one of:
              • genre: takes the following operators (<op>):
                • = or == for testing equality
                • != for testing unequality
                • ~ for testing inclusion (test if the string is contained in)
              • artist: same operators as genre
              • album: same operators as genre
              • song: same operators as genre
              • rate: for filtering on song's rating. Note that unrated songs are considered as 0-rated
                • = or == for testing strict equality
                • != for testing unequality
                • > or >= for strict or non-strict superiority
                • < or <= for strict or non-strict inferiority
              For instance, if you search for {rate>=4}{genre~rock}, you would get all rock/poprock/rock alt... songs rated 4 at least.
            You can write any combination of those 3 search methods.

            By the way, did you know that you can select multiple items from the library by control-clicking? Yeah, you just need to have a keyboard with "ctrl" key...
